## 3. Architecture

### 3.1 Document Storage

```
Upload:   Client → POST /documents/ → Validate PDF → SHA-256 → DB metadata + filesystem bytes
Verify:   Client → GET /documents/{id}/verify → Read filesystem → Recompute SHA-256 → Compare
Download: Client → GET /documents/{id}/download → Read filesystem → Return PDF stream
```

| Storage Layer | What | Where |
|--------------|------|-------|
| PostgreSQL | Metadata (id, filename, hash, case, uploader, timestamp) | `documents` table |
| Filesystem | Raw PDF bytes | `data/documents/{id}.pdf` |

### 3.2 Report Generation

```
Report:   Client → GET /reports/audit or /reports/case/{id}
          → Authenticate + authorize
          → Query DB for data
          → reportlab generates PDF in memory
          → Log download in audit trail
          → Return PDF stream (application/pdf)
```

Reports include a disclaimer footer: *"This report is an informational snapshot generated by the SCFCA system. It does not constitute authorization for any custody action."* (FR-26)
